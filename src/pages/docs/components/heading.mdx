    The Heading components provide five levels of semantic headings (H1-H5) with consistent styling and optional subtitles. Each heading level has appropriate font sizing and spacing for clear visual hierarchy.

<Section>
    ### Props

    All heading components (Heading1-5) share the same props:

    | Prop | Type | Default | Description |
    |------|------|---------|-------------|
    | title | string | - | Heading text (alternative to children) |
    | subtitle | string | - | Optional subtitle text |
    | className | string | - | Additional CSS classes for the heading element |
    | subtitleClass | string | - | Additional CSS classes for the subtitle |
    | wrapperClass | string | - | Additional CSS classes for the wrapper div |
    | children | ReactNode | - | Custom heading content (alternative to title prop) |

    ### Accessibility
    - Uses semantic HTML heading elements (h1-h5)
    - Maintains proper heading hierarchy
    - Bold font weight for clear visual distinction
    - Subtitle uses muted color for visual hierarchy
    - Tight tracking for improved readability

    ### Best Practices
    - Use Heading1 for page titles
    - Maintain logical heading hierarchy (don't skip levels)
    - Use subtitles to provide context without cluttering the main heading
    - Use children when you need custom styling or complex content
    - Keep headings concise and descriptive
    - Don't overuse subtitles - only when they add meaningful context
</Section>
