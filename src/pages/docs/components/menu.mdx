<Section>
    ### Overview
    The Menu component provides a dropdown menu for actions, navigation, and contextual options. It's built on Base UI Menu primitives and includes support for custom triggers, keyboard navigation, and focus management.

    The Menu component is composed of four parts:
    - **Menu**: The root wrapper component
    - **MenuTrigger**: The button or element that opens the menu
    - **MenuContent**: The dropdown container for menu items
    - **MenuItem**: Individual items within the menu
</Section>

    MenuContent automatically positions itself relative to the trigger using Base UI's positioning system. It includes a backdrop, shadow, and smooth animations.
</Section>

<Section>
    ### Props

    #### Menu
    Accepts all props from the Base UI Menu.Root component.

    | Prop | Type | Default | Description |
    |------|------|---------|-------------|
    | className | string | - | Additional CSS classes |

    #### MenuTrigger
    Accepts all props from the Base UI Menu.Trigger component.

    | Prop | Type | Default | Description |
    |------|------|---------|-------------|
    | className | string | - | Additional CSS classes |

    #### MenuContent
    Accepts all props from the Base UI Menu.Portal, Menu.Positioner, and Menu.Popup components.

    | Prop | Type | Default | Description |
    |------|------|---------|-------------|
    | className | string | - | Additional CSS classes |

    #### MenuItem
    Accepts all props from the Base UI Menu.Item component.

    | Prop | Type | Default | Description |
    |------|------|---------|-------------|
    | className | string | - | Additional CSS classes |

    ### Accessibility
    - Keyboard navigation with arrow keys
    - Escape key closes the menu
    - Focus management when opening/closing
    - Click outside to close
    - Proper ARIA attributes from Base UI

    ### Best Practices
    - Use Menu for contextual actions and navigation
    - Keep menu items concise and actionable
    - Use consistent trigger patterns (buttons, avatars, icons)
    - Avoid nesting menus more than one level deep
    - Consider grouping related items with visual separators
