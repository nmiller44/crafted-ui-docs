    <Section title="Overview">
        The DescList component displays key-value pairs in a structured description list format using semantic HTML (`<dl>`, `<dt>`, `<dd>`). It's ideal for presenting metadata, product details, user profiles, or any structured information where labels and values need to be clearly associated.
        
        The component uses a responsive grid layout that adapts from single column on mobile to a 12-column grid on desktop, making it easy to create complex layouts with varying column spans.
    </Section>

    <Section title="Components">
        ### DescList
        Main wrapper component for the description list.
        
        | Prop | Type | Default | Description |
        | --- | --- | --- | --- |
        | title | string | - | Optional title displayed in the header |
        | subtitle | string | - | Optional subtitle displayed below the title |
        | wrapperClassName | string | - | Additional classes for the outer wrapper div |
        | className | string | - | Additional classes for the `<dl>` element |
        
        ### DescListItem
        Individual key-value pair in the description list.
        
        | Prop | Type | Default | Description |
        | --- | --- | --- | --- |
        | label | string | - | Label text displayed as the key |
        | value | React.ReactNode | - | Value content, can be string or React elements |
        | className | string | - | Additional classes (use for grid column spans) |
        | children | React.ReactNode | - | Alternative to `value` for custom content |
        
        **Note:** By default, each DescListItem spans the full width (`col-span-full`). Use `className` with Tailwind grid utilities like `md:col-span-6` or `md:col-span-4` to create multi-column layouts.
        
        ### DescListHeader
        Header component with title and subtitle (automatically included when using `title` prop on DescList).
        
        | Prop | Type | Default | Description |
        | --- | --- | --- | --- |
        | title | string | - | Header title |
        | subtitle | string | - | Header subtitle |
        | className | string | - | Additional classes |
        
        ### DescListLabel
        Label component for the key in key-value pairs (automatically used by DescListItem).
        
        | Prop | Type | Default | Description |
        | --- | --- | --- | --- |
        | children | React.ReactNode | - | Label content |
        | className | string | - | Additional classes |
    </Section>

    <Section title="Accessibility">
        - Uses semantic HTML (`<dl>`, `<dt>`, `<dd>`) for proper structure
        - Screen readers announce the relationship between terms and definitions
        - Responsive grid layout maintains readability across devices
        - Follows WCAG 2.1 AA guidelines
    </Section>

    <Section title="Best Practices">
        ### When to Use
        - Displaying metadata or specifications
        - User profile information
        - Product details and attributes
        - Configuration settings
        - Any structured key-value data
        
        ### Column Span Guidelines
        - Short values (numbers, dates): `md:col-span-3` or `md:col-span-4`
        - Medium values (names, categories): `md:col-span-6`
        - Long descriptions: `col-span-full` (default)
        - Adjust based on content length and layout needs
        
        ### Content Patterns
        - Keep labels concise (1-3 words when possible)
        - Use consistent terminology across related lists
        - Consider using badges for status indicators
        - Group related items visually using column spans
    </Section>
