    ### Overview
    The Table component provides a structured way to display data in rows and columns. It's composed of six parts that work together to create accessible, styled tables.

    The Table component parts:
    - **Table**: The root wrapper component
    - **TableHead**: The table header section (automatically wraps content in a `<tr>`)
    - **TableHeadCell**: Individual header cells
    - **TableBody**: The table body section
    - **TableRow**: Individual rows in the body
    - **TableCell**: Individual data cells
</Section>

<Section>
    ### Props

    #### Table
    Accepts all props from the native `<table>` element.

    | Prop | Type | Default | Description |
    |------|------|---------|-------------|
    | className | string | - | Additional CSS classes |

    #### TableHead
    Accepts all props from the native `<thead>` element. Automatically wraps children in a `<tr>`.

    | Prop | Type | Default | Description |
    |------|------|---------|-------------|
    | className | string | - | Additional CSS classes |

    #### TableHeadCell
    Accepts all props from the native `<th>` element.

    | Prop | Type | Default | Description |
    |------|------|---------|-------------|
    | className | string | - | Additional CSS classes |
    | colspan | number | - | Number of columns to span |

    #### TableBody
    Accepts all props from the native `<tbody>` element.

    | Prop | Type | Default | Description |
    |------|------|---------|-------------|
    | className | string | - | Additional CSS classes |

    #### TableRow
    Accepts all props from the native `<tr>` element.

    | Prop | Type | Default | Description |
    |------|------|---------|-------------|
    | className | string | - | Additional CSS classes |

    #### TableCell
    Accepts all props from the native `<td>` element.

    | Prop | Type | Default | Description |
    |------|------|---------|-------------|
    | className | string | - | Additional CSS classes |
    | colspan | number | - | Number of columns to span |

    ### Accessibility
    - Semantic HTML table elements for proper structure
    - TableHead provides proper `<thead>` semantics
    - Column headers use `<th>` elements for screen readers
    - Left-aligned text for better readability

    ### Best Practices
    - Use TableHead for column headers
    - Place tables inside Card components for visual structure
    - Use colspan sparingly for complex layouts
    - Add hover states with `hover:bg-muted/50` on TableRow
    - Make key columns bold with `font-medium` or `font-bold`
    - Keep column widths consistent for better scanning
</Section>
