    <Section title="Props">
        The `Empty` component accepts the following props:
        
        | Prop | Type | Default | Description |
        | --- | --- | --- | --- |
        | title | string | - | Primary heading for the empty state |
        | subtitle | string | - | Supporting text below the title |
        | children | React.ReactNode | - | Custom content (used instead of title/subtitle when title is omitted) |
        | className | string | - | Additional CSS classes for customization |
        
        ### Behavior
        - When `title` is provided, it displays with optional `subtitle` using the Heading3 component
        - When `title` is omitted, children content is displayed instead
        - Cannot use both `title` and `children` simultaneously - title takes precedence
        
        ### Styling Features
        - Centered layout with max-width constraint
        - Dashed border in muted color for visual distinction
        - Rounded corners for modern appearance
        - Generous padding for comfortable spacing
        - Muted text colors for empty state aesthetics
    </Section>

    <Section title="Accessibility">

        - Clear messaging helps users understand the empty state

    <Section title="Best Practices">
        ### Messaging Guidelines
        - **Be specific**: Explain why the state is empty ("No search results" vs "Empty")
        - **Provide guidance**: Suggest what users can do next
        - **Stay positive**: Frame messages constructively
        - **Be concise**: Keep titles short, use subtitle for details
        
        ### When to Use
        - Empty search or filter results
        - New user states (no data yet)
        - Fully cleared lists or collections
        - Error states with no fallback content
        
        ### Action Buttons
        - Primary action: Most common next step (e.g., "Add Item")
        - Secondary action: Alternative options (e.g., "Clear Filters")
        - Place buttons below the message for natural flow
        
        ### Visual Enhancements
        - Consider adding icons or emoji for personality
        - Use illustrations for important empty states
        - Keep custom content centered and balanced
    </Section>
